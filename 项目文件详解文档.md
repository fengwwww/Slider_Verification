### `backend/.../common/Result.java`

**作用**：统一 API 响应格式封装类，所有接口返回的数据结构都用它包装。

```java
@Data
public class Result {
    private int code;        // 状态码：200=成功，400=参数错误，403=验证码失败
    private String message;  // 提示信息
    private Object data;     // 返回的数据（可选）

    public static Result success(String message) { ... }
    public static Result error(int code, String message) { ... }
}
```

**说明**：前端收到的 JSON 永远是 `{ code, message, data }` 格式，方便前端统一处理。比如：
- 注册成功 → `{ code: 200, message: "注册成功" }`
- 用户名重复 → `{ code: 400, message: "用户名已存在" }`
- 验证码失败 → `{ code: 403, message: "滑块验证未通过" }`

**定位**：后端所有 Controller 的返回值都用它，保证接口格式统一。

### `backend/.../config/CaptchaProperties.java`

**作用**：把 `application.yml` 中的 `captcha.*` 配置映射到 Java 对象，方便代码中读取。

```java
@Data

public class CaptchaProperties {
```

**说明**：
- `@ConfigurationProperties(prefix = "captcha")`：自动把 yml 中 `captcha.captcha-app-id` 映射到 `captchaAppId` 字段（Spring Boot 松散绑定：kebab-case 转 camelCase）
- `@Component`：注册为 Spring Bean，可以在其它类中 `@Autowired` 注入

**定位**：配置与代码之间的桥梁。`CaptchaService` 通过它读取验证码凭证和开关状态。

### `backend/.../config/CorsConfig.java`

**作用**：跨域资源共享（CORS）配置，允许前端跨域访问后端接口。

```java
@Configuration
public class CorsConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
                .allowedOriginPatterns("*")
                .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
                .allowedHeaders("*")
                .allowCredentials(true);
    }
}
```

**说明**：前端运行在 `localhost:5173`，后端在 `localhost:8080`，端口不同属于跨域。浏览器默认阻止跨域请求，这里配置允许所有来源的跨域访问。

> 注意：实际开发中前端通过 Vite 代理（`vite.config.js`）转发请求，浏览器看到的请求是同源的。这个 CORS 配置作为兜底，确保直连后端也能正常访问。

**定位**：解决前后端跨域问题的安全配置。

### `backend/.../controller/UserController.java`

**作用**：用户相关的 REST API 接口控制器，处理登录/注册 HTTP 请求。

**核心职责**：
1. 接收前端发来的 HTTP 请求（含 username、password、ticket、randstr）
2. **先调用 CaptchaService 校验验证码票据**（验证失败直接返回 403）
3. 验证通过后再调用 UserService 执行注册/登录业务逻辑
4. 返回统一格式的 Result 响应

```java
@PostMapping("/register")
public Result register(@RequestBody User user, HttpServletRequest request) {
    // ① 先校验滑动验证码票据
    if (!captchaService.verify(user.getTicket(), user.getRandstr(), getClientIp(request))) {
        return Result.error(403, "滑块验证未通过，请重试");
    }
    // ② 验证通过，执行注册
    boolean ok = userService.register(user);
    return ok ? Result.success("注册成功") : Result.error(400, "用户名已存在");
}
```

**还包含**：`getClientIp()` 方法，从 HTTP 请求头提取客户端真实 IP（兼容多级代理），传给腾讯云 API 做风控校验。

**定位**：HTTP 请求的入口，验证码校验的拦截点。业务逻辑不写在这里，而是委托给 Service 层。

---

### `backend/.../entity/User.java`

**作用**：用户实体类，JPA 映射数据库 `users` 表。

```java
@Data
@Entity
@Table(name = "users")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;            // 主键，自增

    private String username;     // 用户名
    private String password;     // 密码

    @Transient
    private String ticket;       // 验证码票据（不存库）

    @Transient
    private String randstr;      // 验证码随机串（不存库）
}
```

**说明**：
- `@Entity`：标识为 JPA 实体，对应数据库表
- `@Table(name = "users")`：指定表名
- `@Id` + `@GeneratedValue`：主键自增
- `@Transient`：标记 `ticket` 和 `randstr` 不映射数据库列。这两个字段只用于接收前端传来的验证码参数，验证完就丢弃，不落库
- `@Data`（Lombok）：自动生成 getter/setter/toString 等

**定位**：既是数据库表的映射，又兼做请求 DTO（接收前端的 username/password/ticket/randstr）。

---

### `backend/.../repository/UserRepository.java`

**作用**：数据库操作接口，提供对 `users` 表的增删改查方法。

```java
public interface UserRepository extends JpaRepository<User, Long> {
    User findByUsername(String username);
}
```

**说明**：
- 继承 `JpaRepository<User, Long>`：自动获得 `save()`、`findById()`、`findAll()`、`delete()` 等标准 CRUD 方法
- `findByUsername(String username)`：Spring Data JPA 的方法名约定查询，自动生成 `SELECT * FROM users WHERE username = ?` 的 SQL

**定位**：数据访问层（DAO），UserService 通过它操作数据库。

---

### `backend/.../service/CaptchaService.java`

**作用**：腾讯云滑动验证码票据校验服务——后端验证的核心。

**核心逻辑**：
1. 检查 `enabled` 开关，关闭时直接放行（开发联调用）
2. 检查 ticket/randstr 是否为空
3. 用 SecretId + SecretKey 构建 CAM 凭证
4. 创建腾讯云 CaptchaClient
5. 组装请求参数（CaptchaType=9、ticket、randstr、userIp、captchaAppId、appSecretKey）
6. 调用 `DescribeCaptchaResult` API
7. 判断返回的 `CaptchaCode == 1`（1=验证通过）
8. 异常时返回 false（安全优先，fail-closed）

**定位**：连接后端与腾讯云的桥梁。没有它，黑客可以伪造 ticket 绕过验证。这是整个安全方案的关键。

---

### `backend/.../service/UserService.java`

**作用**：用户注册和登录的业务逻辑实现。

```java
@Service
public class UserService {
    @Autowired
    private UserRepository userRepository;

    public boolean register(User user) {
        if (userRepository.findByUsername(user.getUsername()) != null) {
            return false;  // 用户名已存在
        }
        userRepository.save(user);
        return true;
    }

    public boolean login(User user) {
        User dbUser = userRepository.findByUsername(user.getUsername());
        if (dbUser == null) return false;
        return dbUser.getPassword().equals(user.getPassword());
    }
}
```

**说明**：
- **注册**：先查用户名是否已存在，不存在才保存
- **登录**：查用户名，比对密码是否一致

> 注意：本项目密码是明文存储和比对，仅作教学演示。生产环境应该用 BCrypt 等算法加密存储。

**定位**：纯粹的业务逻辑层，不关心验证码（验证码在 Controller 层已拦截）。只负责用户数据的增删改查判断。

### `frontend/src/captcha.js`

**作用**：腾讯云滑块验证的封装模块，把回调式 API 转成 Promise。

```js
export const CAPTCHA_APP_ID = '你的CaptchaAppId'

export function showCaptcha() {
  return new Promise((resolve, reject) => {
    const captcha = new window.TencentCaptcha(CAPTCHA_APP_ID, (res) => {
      if (res.ret === 0 && res.ticket && res.randstr) {
        resolve({ ticket: res.ticket, randstr: res.randstr })
      } else {
        reject(new Error('用户未完成滑块验证'))
      }
    }, {})
    captcha.show()
  })
}
```

**说明**：
- `CAPTCHA_APP_ID`：验证码应用 ID，标识是哪个验证（必须用 Web 类型的）
- `showCaptcha()`：弹出滑块验证弹窗，用户滑过后 resolve 返回 `{ ticket, randstr }`，用户关闭或失败时 reject
- 用 Promise 封装的原因：腾讯云 SDK 是回调式的，封装成 Promise 后业务代码可以用 `await showCaptcha()` 同步写法

**定位**：前端验证码的唯一入口，Login.vue 和 Register.vue 都通过它弹出滑块。

```js
})

export const login = (data) => api.post('/user/login', data)
export const register = (data) => api.post('/user/register', data)
```

**说明**：
- `baseURL: '/api'`：所有请求自动加 `/api` 前缀，配合 Vite 代理转发到后端
- `login(data)`：POST `/api/user/login`，data 包含 username、password、ticket、randstr
- `register(data)`：POST `/api/user/register`，参数同上

**定位**：前端所有后端调用的统一出口，集中管理接口地址和超时配置。

**作用**：Vue Router 路由配置，定义 URL 与页面组件的对应关系。

```js
const routes = [
```

**说明**：
- 访问 `http://localhost:5173/` → 自动跳转 `/login`
- 访问 `/login` → 渲染 Login.vue
- 访问 `/register` → 渲染 Register.vue
- 访问 `/home` → 渲染 Home.vue

**定位**：前端的"导航地图"，决定 URL 对应显示哪个页面。

### `frontend/src/views/Login.vue`

**作用**：登录页面组件。

**核心逻辑**（`handleLogin` 方法）：
1. 表单校验：用户名密码不能为空
2. 调用 `showCaptcha()` 弹出滑块验证，拿到 ticket + randstr
3. 调用 `login()` 接口发送请求（username + password + ticket + randstr）
4. 后端返回 200 → 存 localStorage，跳转首页
5. 后端返回非 200 → 显示错误信息

**定位**：用户登录的交互入口，滑块验证的第一个调用方。

---

### `frontend/src/views/Register.vue`

**作用**：注册页面组件。

**核心逻辑**（`handleRegister` 方法）：
1. 表单校验：用户名、密码、确认密码不能为空
2. 密码一致性检查：两次输入必须一致
3. 调用 `showCaptcha()` 弹出滑块验证
4. 调用 `register()` 接口发送请求
5. 注册成功 → 1秒后跳转登录页

**定位**：用户注册的交互入口，逻辑结构与 Login.vue 一致，多了确认密码校验。

---

### `frontend/src/views/Home.vue`

**作用**：登录成功后的首页。

```vue
<template>
  <h2>欢迎，{{ username }}！</h2>
  <button @click="logout">退出登录</button>
</template>
```

**说明**：
- 从 `localStorage` 读取用户名显示
- 「退出登录」按钮清空 localStorage 并跳转回登录页

**定位**：登录成功的"终点"页面，纯展示用途，不涉及验证码逻辑。

## 四、文件关系与数据流

以用户注册为例，各文件的调用关系：

```
用户操作
  │
  ▼
Register.vue ────→ captcha.js ────→ TJCaptcha.js (index.html 加载)
  │  (弹出滑块)        (封装 Promise)        (腾讯云 SDK)
  │                                            │
  │                                    返回 ticket + randstr
  │                                            │
  ▼                                            │
api/index.js ────→ Vite 代理 ────→ 后端 8080   │
  (axios 请求)      (vite.config.js)           │
                                                │
                              ▼                │
                    UserController.java         │
                      (接收请求)                 │
                          │                     │
                    CaptchaService.java ◀──────┘
                      (校验票据)
                          │ 调用腾讯云 API
                          ▼
                    CaptchaProperties.java
                      (读取配置凭证)
                          │
                    UserService.java
                      (执行注册)
                          │
                    UserRepository.java
                      (写入 MySQL)
```

| 层 | 文件 | 职责 |
|----|------|------|

| 验证码 | captcha.js | 封装滑块验证 |

| 配置 | application.yml / CaptchaProperties.java / CorsConfig.java | 运行参数、跨域 |
| 通用 | Result.java / LoginApplication.java | 统一响应、启动入口 |
