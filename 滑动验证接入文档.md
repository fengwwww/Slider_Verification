# 腾讯云滑动验证码接入文档

> 本文档详细记录了如何将腾讯云滑动验证码（滑动验证 2.0）接入到 Vue 3 + Spring Boot 的登录注册项目中，涵盖前后端每一处改动、代码原理和完整验证流程。

---

### 设计原则：两层验证

| 层级 | 作用 | 说明 |
|------|------|------|
| **前端滑块** | 人机识别 | 弹出滑块让用户滑动，通过后拿到 `ticket` + `randstr` 票据 |
| **后端票据校验** | 防伪造 | 后端拿票据调用腾讯云 API 二次验证，防止黑客直接构造请求绕过滑块 |

> ⚠️ **如果只做前端验证、不做后端校验**：黑客可以直接用脚本构造 HTTP 请求（带随便填的 ticket），绕过滑块直接注册/登录。后端票据校验是安全的关键。

## 二、前置准备：获取腾讯云凭证

接入需要 **4 项凭证**，分两个控制台获取：

### 2.1 验证码控制台

地址：https://console.cloud.tencent.com/captcha

1. 进入「验证管理」→ 新建验证
2. 客户端类型选择 **Web**
3. 在「基础配置」页面获取：
   - **CaptchaAppId**（如 `你的CaptchaAppId`）
   - **AppSecretKey**（如 `你的AppSecretKey`）

### 2.2 CAM 访问管理

地址：https://console.cloud.tencent.com/cam/capi

1. 点击「新建密钥」
2. 创建时弹窗会同时显示：
   - **SecretId**（如 `你的SecretId`）
   - **SecretKey**（如 `你的SecretKey`）

> ⚠️ SecretKey 只在创建时显示一次，关闭弹窗后无法再查看。务必保存。

### 凭证对照表

| 凭证 | 用在哪 | 来源 |
|------|--------|------|
| `CaptchaAppId` | 前端 captcha.js + 后端 application.yml | 验证码控制台 |
| `AppSecretKey` | 后端 application.yml | 验证码控制台 |
| `SecretId` | 后端 application.yml | CAM 密钥管理 |
| `SecretKey` | 后端 application.yml | CAM 密钥管理 |

## 三、前端实现

前端共改动/新增 **4 个文件**，核心思路：加载腾讯云 JS SDK → 封装成 Promise → 在登录注册按钮中调用 → 把票据随请求发给后端。

### 3.1 加载验证码 SDK —— `frontend/index.html`

```html
<head>
  <!-- 腾讯云验证码程序依赖（必须），请勿本地缓存或规避加载 -->
  <script src="https://turing.captcha.qcloud.com/TJCaptcha.js"></script>
</head>
```

**原理**：`TJCaptcha.js` 是腾讯云提供的验证码 SDK，加载后会在 `window` 上挂载全局构造函数 `TencentCaptcha`。这个 JS 必须从腾讯云 CDN 加载，不能本地缓存或规避加载，否则验证码无法正常更新，对抗能力无法保证。

### 3.2 封装滑块验证 —— `frontend/src/captcha.js`

```js
export const CAPTCHA_APP_ID = '你的CaptchaAppId'  // ← 填入你的 CaptchaAppId

/**
 * 弹出滑块验证
 * 成功返回 { ticket, randstr }，用户关闭或失败时 reject
 */
export function showCaptcha() {
  return new Promise((resolve, reject) => {
    if (typeof window.TencentCaptcha !== 'function') {
      reject(new Error('验证码脚本未加载，请检查网络'))
      return
    }
    try {
      const captcha = new window.TencentCaptcha(CAPTCHA_APP_ID, (res) => {
        if (res.ret === 0 && res.ticket && res.randstr) {
          resolve({ ticket: res.ticket, randstr: res.randstr })
        } else {
          reject(new Error('用户未完成滑块验证'))
        }
      }, {})
      captcha.show()
    } catch (e) {
      reject(e)
    }
  })
}
```

**逐步解析**：

| 代码 | 作用 |
|------|------|
| `export const CAPTCHA_APP_ID` | 验证码应用 ID，告诉腾讯云这是哪个验证（必须用 Web 类型的） |
| `new window.TencentCaptcha(appId, callback, options)` | 创建验证码实例，传入 AppId 和回调函数 |
| `captcha.show()` | 弹出滑块验证弹窗 |
| `res.ret === 0` | 验证成功标志（0=成功，2=用户主动关闭） |
| `res.ticket` | 验证票据，后端校验用 |
| `res.randstr` | 随机串，后端校验用 |
| `Promise` 封装 | 把回调式 API 转成 Promise，方便 `await` 调用 |

**为什么要封装成 Promise**：腾讯云的 SDK 是回调式的（传入 callback 函数），但 Vue 3 中我们习惯用 `async/await`。用 `new Promise` 把回调包装成 Promise 后，就能在业务代码中用 `await showCaptcha()` 同步写法，代码更清晰。

### 3.3 登录页接入 —— `frontend/src/views/Login.vue`

```js
import { showCaptcha } from '../captcha'

const handleLogin = async () => {
  // 1. 基本表单校验
  if (!form.username || !form.password) {
    msg.value = '请填写完整信息'
    return
  }
  // 2. 先完成滑块验证，拿到 ticket 与 randstr
  let captcha
  try {
    captcha = await showCaptcha()   // ← 弹滑块，用户滑过后返回票据
  } catch (e) {
    msg.value = '请完成滑块验证'      // 用户关闭或失败
    return
  }
  // 3. 把票据随账号密码一起发给后端
  try {
    const res = await login({
      ...form,                       // username + password
      ticket: captcha.ticket,        // 验证票据
      randstr: captcha.randstr       // 随机串
    })
    // 4. 处理后端返回
    success.value = res.data.code === 200
    msg.value = res.data.message
    if (success.value) {
      localStorage.setItem('username', form.username)
      setTimeout(() => router.push('/home'), 500)
    }
  } catch (e) {
    msg.value = '登录失败，请检查网络或后端服务'
  }
}
```

**关键时序**：表单校验 → 滑块验证（弹窗） → 拿到票据 → 发请求（带票据） → 后端校验票据 → 返回结果。

> 用户必须先滑完滑块才能提交请求，不存在「跳过滑块直接提交」的可能。

### 3.4 注册页接入 —— `frontend/src/views/Register.vue`

注册页逻辑与登录页完全一致，唯一区别是多了一个「确认密码」校验：

```js
// 注册页在滑块验证前多了密码一致性检查
if (form.password !== form.confirm) {
  msg.value = '两次密码不一致'
  return
}
// 之后同样：await showCaptcha() → 发请求（带 ticket + randstr）
```

### 3.5 API 层 —— `frontend/src/api/index.js`

```js
const api = axios.create({
  baseURL: '/api',      // Vite 代理到后端 8080
  timeout: 10000
})

export const login = (data) => api.post('/user/login', data)
export const register = (data) => api.post('/user/register', data)
```

`data` 中包含 `username`、`password`、`ticket`、`randstr` 四个字段，通过 Vite 代理转发给后端。

### 3.6 前端文件清单

```
frontend/
├── index.html                  ← 加载 TJCaptcha.js
└── src/
    ├── captcha.js              ← 封装 showCaptcha()
    ├── api/index.js            ← axios 请求封装
    └── views/
        ├── Login.vue           ← 登录页（调 showCaptcha）
        └── Register.vue        ← 注册页（调 showCaptcha）
```

## 四、后端实现

后端共改动/新增 **6 个文件**，核心思路：加 SDK 依赖 → 配置凭证 → 写票据校验服务 → 在 Controller 中拦截校验。

### 4.1 添加 SDK 依赖 —— `backend/pom.xml`

```xml
<!-- 腾讯云验证码 SDK -->
<dependency>
    <groupId>com.tencentcloudapi</groupId>
    <artifactId>tencentcloud-sdk-java-captcha</artifactId>
    <version>3.1.1417</version>
</dependency>
```

**原理**：这个 SDK 封装了调用腾讯云 `DescribeCaptchaResult` API 的全部逻辑（包括 CAM 签名算法、HTTP 请求、响应解析），不用自己手写签名和 HTTP 调用。SDK 版本 `3.1.1417` 是 Maven Central 上的最新稳定版。

### 4.2 配置凭证 —— `backend/src/main/resources/application.yml`

```yaml
# 腾讯云滑动验证码配置
captcha:
  enabled: true                              # 后端票据校验开关
  captcha-app-id: 你的CaptchaAppId                 # 验证码应用ID
  app-secret-key: 你的AppSecretKey # 验证码应用密钥
  secret-id: 你的SecretId  # CAM SecretId
  secret-key: 你的SecretKey      # CAM SecretKey
  region: ap-guangzhou                       # API地域
```

| 参数 | 说明 |
|------|------|
| `enabled` | `false`=跳过后端校验（开发联调），`true`=调用腾讯云API校验（生产） |
| `captcha-app-id` | 验证码应用 ID（验证码控制台获取） |
| `app-secret-key` | 验证码应用密钥（验证码控制台获取） |
| `secret-id` | CAM 访问密钥 ID（CAM 密钥管理获取） |
| `secret-key` | CAM 访问密钥（CAM 密钥管理获取） |
| `region` | 腾讯云 API 地域，captcha 默认用 `ap-guangzhou` |

### 4.3 配置属性绑定 —— `CaptchaProperties.java`

```java
@Data
@Component
@ConfigurationProperties(prefix = "captcha")
public class CaptchaProperties {
    private boolean enabled = false;
    private String captchaAppId;
    private String appSecretKey;
    private String secretId;
    private String secretKey;
    private String region = "ap-guangzhou";
}
```

**原理**：`@ConfigurationProperties(prefix = "captcha")` 会自动把 `application.yml` 中 `captcha.*` 下的配置映射到这个类的字段上。比如 `captcha.captcha-app-id` 自动映射到 `captchaAppId`（Spring Boot 的松散绑定规则：kebab-case → camelCase）。这样在代码中就能用 `captchaProperties.getCaptchaAppId()` 拿到配置值，而不是手动读取 yml。

### 4.4 票据校验服务 —— `CaptchaService.java`（核心）

```java
@Slf4j
@Service
public class CaptchaService {

    @Autowired
    private CaptchaProperties captchaProperties;

    public boolean verify(String ticket, String randstr, String userIp) {
        // ① 未开启后端校验，直接放行（开发联调阶段）
        if (!captchaProperties.isEnabled()) {
            return true;
        }
        // ② 票据参数为空，直接拒绝
        if (ticket == null || ticket.isBlank() || randstr == null || randstr.isBlank()) {
            return false;
        }
        try {
            // ③ 构建CAM凭证（用 SecretId + SecretKey 签名）
            Credential cred = new Credential(
                    captchaProperties.getSecretId(),
                    captchaProperties.getSecretKey()
            );
            // ④ 构建客户端
            HttpProfile httpProfile = new HttpProfile();
            httpProfile.setEndpoint("captcha.tencentcloudapi.com");
            ClientProfile clientProfile = new ClientProfile();
            clientProfile.setHttpProfile(httpProfile);
            CaptchaClient client = new CaptchaClient(
                    cred, captchaProperties.getRegion(), clientProfile
            );
            // ⑤ 组装请求参数
            DescribeCaptchaResultRequest req = new DescribeCaptchaResultRequest();
            req.setCaptchaType(9L);           // 固定值：滑动验证2.0
            req.setTicket(ticket);            // 前端传来的票据
            req.setRandstr(randstr);          // 前端传来的随机串
            req.setUserIp(userIp);            // 客户端真实IP
            req.setCaptchaAppId(Long.parseLong(captchaProperties.getCaptchaAppId()));
            req.setAppSecretKey(captchaProperties.getAppSecretKey());
            // ⑥ 发起校验（SDK 内部完成 CAM 签名 + HTTP 调用）
            DescribeCaptchaResultResponse resp = client.DescribeCaptchaResult(req);
            // ⑦ CaptchaCode = 1 表示验证通过
            boolean ok = resp.getCaptchaCode() != null && resp.getCaptchaCode() == 1L;
            if (!ok) {
                log.warn("验证码校验失败: code={}, msg={}",
                        resp.getCaptchaCode(), resp.getCaptchaMsg());
            }
            return ok;
        } catch (Exception e) {
            log.error("验证码票据校验异常", e);
            return false;  // 校验异常时拒绝请求（安全优先）
        }
    }
}
```

**逐段解析**：

| 步骤 | 代码 | 说明 |
|------|------|------|
| ① 开关判断 | `isEnabled()` | `enabled=false` 时直接 return true，方便开发联调（前端仍弹滑块，但后端不验票据） |
| ② 空值检查 | `ticket.isBlank()` | 防止黑客不传票据就请求 |
| ③ CAM 凭证 | `new Credential(secretId, secretKey)` | 腾讯云所有 API 都需要 CAM 认证，用 SecretId/SecretKey 生成签名 |
| ④ 客户端 | `new CaptchaClient(cred, region, profile)` | SDK 客户端，指定 API 端点和地域 |
| ⑤ 请求参数 | `setCaptchaType(9L)` | **9 = 滑动验证 2.0**（固定值，不能改） |
| ⑤ 请求参数 | `setTicket / setRandstr` | 前端滑块通过后返回的票据，后端拿去验真伪 |
| ⑤ 请求参数 | `setUserIp` | 客户端 IP，腾讯云用于风控（同一 IP 短时间大量验证会被拦截） |
| ⑥ 发起校验 | `client.DescribeCaptchaResult(req)` | SDK 内部完成 TC3-HMAC-SHA256 签名 + HTTP POST |
| ⑦ 结果判断 | `getCaptchaCode() == 1L` | 1=通过，其它值=失败（如票据过期、票据伪造、IP 不匹配等） |

**为什么需要 `userIp`**：腾讯云会校验「发起滑块验证的 IP」和「提交票据校验的 IP」是否一致。如果黑客盗用了别人的票据（从别处抓包），IP 不匹配，校验会失败。

### 4.5 实体类加瞬态字段 —— `User.java`

```java
@Data
@Entity
@Table(name = "users")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String username;
    private String password;

    /** 验证码票据（不落库，仅用于接收前端参数） */
    @Transient
    private String ticket;

    /** 验证码随机串（不落库，仅用于接收前端参数） */
    @Transient
    private String randstr;
}
```

**原理**：

- 前端发来的 JSON 是 `{ username, password, ticket, randstr }`，后端用 `@RequestBody User user` 接收
- Jackson 自动把 JSON 的 4 个字段映射到 User 对象的 4 个属性
- `@Transient`（`jakarta.persistence.Transient`）告诉 JPA：这两个字段不映射数据库列，不会建表也不会存库
- 这样既复用了现有的 User 实体接收请求参数，又不需要单独写 DTO

> `ticket` 和 `randstr` 是一次性的验证凭证，验证完就作废了，不需要存数据库。

### 4.6 Controller 拦截校验 —— `UserController.java`

```java
@RestController
@RequestMapping("/api/user")
public class UserController {

    @Autowired
    private UserService userService;

    @Autowired
    private CaptchaService captchaService;

    @PostMapping("/register")
    public Result register(@RequestBody User user, HttpServletRequest request) {
        // 先校验滑动验证码票据
        if (!captchaService.verify(
                user.getTicket(), user.getRandstr(), getClientIp(request))) {
            return Result.error(403, "滑块验证未通过，请重试");
        }
        boolean ok = userService.register(user);
        return ok ? Result.success("注册成功") : Result.error(400, "用户名已存在");
    }

    @PostMapping("/login")
    public Result login(@RequestBody User user, HttpServletRequest request) {
        // 先校验滑动验证码票据
        if (!captchaService.verify(
                user.getTicket(), user.getRandstr(), getClientIp(request))) {
            return Result.error(403, "滑块验证未通过，请重试");
        }
        boolean ok = userService.login(user);
        return ok ? Result.success("登录成功") : Result.error(400, "用户名或密码错误");
    }

    /** 获取客户端真实IP（兼容多级代理） */
    private String getClientIp(HttpServletRequest request) {
        String ip = request.getHeader("X-Forwarded-For");
        if (ip == null || ip.isEmpty() || "unknown".equalsIgnoreCase(ip)) {
            ip = request.getHeader("Proxy-Client-IP");
        }
        if (ip == null || ip.isEmpty() || "unknown".equalsIgnoreCase(ip)) {
            ip = request.getHeader("WL-Proxy-Client-IP");
        }
        if (ip == null || ip.isEmpty() || "unknown".equalsIgnoreCase(ip)) {
            ip = request.getRemoteAddr();
        }
        if (ip != null && ip.contains(",")) {
            ip = ip.split(",")[0].trim();  // 多级代理取第一个
        }
        return ip;
    }
}
```

**关键点**：

- **验证在前，业务在后**：先 `captchaService.verify()`，通过后才执行 `userService.register/login()`。如果验证失败，直接返回 403，根本不会走到业务逻辑
- `getClientIp()`：从 `HttpServletRequest` 提取客户端真实 IP。优先读 `X-Forwarded-For`（经过 Nginx 等代理时携带），兜底用 `request.getRemoteAddr()`
- 登录和注册两个接口的校验逻辑完全一致

## 五、完整验证流程

以用户点击「注册」为例：

```
用户输入用户名/密码，点击「注册」按钮
         │
         ▼
┌─────────────────────────────────┐
│ 1. 前端表单校验                  │  Register.vue: 检查是否为空、两次密码是否一致
└────────────────┬────────────────┘
                 │ 通过
                 ▼
```

## 六、关键设计决策说明

### 6.1 为什么用 `@Transient` 而不是单独写 DTO？

前端传来的 `{ username, password, ticket, randstr }` 中，`username` 和 `password` 需要存库，`ticket` 和 `randstr` 是一次性验证凭证不需要存。两种方案：

| 方案 | 优点 | 缺点 |
|------|------|------|
| **`@Transient`（选用）** | 改动最小，复用 User 实体 | 验证字段混在实体里 |
| 单独写 DTO | 职责分离更干净 | 多一个类，Controller 要手动拆分字段 |

本项目是教学 Demo，优先简洁，选了 `@Transient`。生产项目可以用 DTO 方案。

### 6.2 为什么 `enabled` 默认 false？

```yaml
captcha:
  enabled: false  # 开发时默认关闭
```

- **开发阶段**：没有真实凭证或不想每次都调 API，设 `false` 后后端跳过校验直接放行，方便调试
- **生产阶段**：设 `true`，后端真正调用腾讯云 API 校验票据

前端滑块无论 `enabled` 是什么都会弹出（这是前端 JS 控制的），区别只在后端是否验真票据。

### 6.3 为什么校验异常时返回 false？

```java
} catch (Exception e) {
    log.error("验证码票据校验异常", e);
    return false;  // 安全优先：异常时拒绝
}
```

安全系统的原则是「失败时拒绝（fail-closed）」。如果校验过程中网络断了、API 超时了，宁可让用户重试，也不能放行一个未验证的请求。

### 6.4 CaptchaType 为什么是 9？

腾讯云 `DescribeCaptchaResult` API 的 `CaptchaType` 参数标识验证码类型：

| 值 | 类型 |
|----|------|
| 9 | 滑动验证 2.0（本项目使用） |
| 其它 | 其它验证码类型 |

这个值是固定写死的，对应你在验证码控制台创建的「滑动验证 2.0」类型。

## 八、安全注意事项

1. **密钥不要提交到公开仓库**：`application.yml` 中的 `secret-key` 和 `app-secret-key` 是敏感信息，建议用环境变量读取或放在 `.gitignore` 的本地配置文件中
2. **`enabled` 生产必须设 true**：否则后端不校验票据，等于没接入验证码
3. **SecretKey 只显示一次**：CAM 密钥创建后 SecretKey 无法再查看，丢失只能新建
4. **CaptchaAppId 必须用 Web 类型**：不能用小程序类型的，会导致数据统计错误
